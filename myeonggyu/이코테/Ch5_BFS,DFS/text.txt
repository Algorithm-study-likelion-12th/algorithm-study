collections 모듈에서 제공하는 deque 자료구조는 데이터를 넣고 빼는 속도가 리스트 자료형에 비해서 효율적이며
queue 라이브러리를 이용하는 것보다 더 간단함
-> 그렇다면 스택 자료구조를 사용하는 경우에도 리스트를 사용하는 것보다 deque 자료구조를 사용하는 것이 더 효율적이지 않을까?
-> 리스트를 사용해서 끝의 데이터를 처리하는 것은 O(1)의 시간복잡도를 가지기 때문에 충분히 효율적이라고 할 수 있음
-> 추가로 리스트는 기본 자료구조로 사용되기 때문에 사용법에 익숙하고 더 직관적이라는 장점이 있음

def bfs(graph, start, vistied):
    queue = deque([start])
위의 코드를 보고 deque에 매개변수로 리스트를 넣어주는 것이 의아하게 느껴짐.
하지만 deque 클래스가 선언되어 있는 collections 패키지를 조회해본 결과 다음과 같이 init이 구현되어 있는 것을 확인할 수 있었음
def __init__(self, iterable=(), maxlen=None):
iterable 객체를 입력 매개변수로 줘야 하지만 start는 int였고 iterable 객체가 아니기 때문에 리스트 객체로 만들어서 입력해야 한다는 것을 알 수 있었음.
다른 예시로 queue = deque('abc') 처럼 string 객체를 입력하는 경우 iterable 객체이기 때문에 리스트로 감싸지 않고 입력할 수 있음.
